# 07 - UI Pages and Components

## Overview

The analytics UI is integrated into the existing Next.js app (`apps/web`), following the same pattern as Sites and Pomerium extensions. It consists of:

1. A dedicated `/analytics` section with multi-tab dashboard
2. A public dashboard route (`/analytics/public/[token]`)

The analytics module is **separate from the existing domain detail page**. Existing server-side HAProxy metrics on `/domains/[id]` remain untouched. The existing `site_analytics` table tracks server-side HAProxy log data (request counts, status codes, response times); the new analytics module tracks client-side JS interactions (pageviews, events, visitor behaviour). Both coexist and serve different purposes.

## Page Specifications

### `/analytics` - Analytics Overview

The landing page for the analytics section. Shows all domains with analytics enabled.

**Layout**:
- Header: "Analytics" title with a brief description
- Stats cards row: Total tracked domains, Total pageviews (last 7d), Total visitors (last 7d)
- Domain list: Table or card grid showing each domain with:
  - Domain hostname
  - Enabled/disabled status badge
  - Sparkline chart (last 7 days of pageviews)
  - Key metrics: pageviews, visitors, bounce rate
  - Link to detailed dashboard
  - "Enable Analytics" button for domains without analytics

**Extension check**: Use `useAnalyticsExtensionEnabled()` hook (follows `usePomeriumExtensionEnabled()` pattern). If not enabled, show an information card explaining how to enable the extension.

### `/analytics/[configId]` - Per-Domain Dashboard

Multi-tab dashboard for a single domain's analytics.

**Tabs**:

| Tab | Component | Content |
|-----|-----------|---------|
| Overview | `analytics-overview.tsx` | Summary cards, traffic chart, top pages, top referrers |
| Pages | `analytics-pages.tsx` | Full pages table with sorting, entry/exit pages |
| Referrers | `analytics-referrers.tsx` | Referrer domains, UTM campaigns/sources/mediums |
| Geography | `analytics-geography.tsx` | Country list with visitor counts (sorted) |
| Devices | `analytics-devices.tsx` | Device type breakdown, browsers, OS |
| Events | `analytics-events.tsx` | Custom events list, event detail drill-down |
| Funnels | `analytics-funnels.tsx` | Funnel definitions and visualisations |
| Real-time | `analytics-realtime.tsx` | Live visitor count, active pages, event stream |
| Settings | `analytics-settings.tsx` | Configuration, embed snippet, public dashboard toggle |

**Period selector**: Shared across all tabs. Options: Last 24h, Last 7d, Last 30d, Last 90d, Custom range.

**Comparison toggle**: Show/hide comparison with previous equivalent period.

**Cross-dimensional filtering**: Add a filter bar/panel to the dashboard that allows filtering by country, device type, browser, OS, referrer domain, UTM source/medium/campaign, and pathname prefix. Filters apply across all tabs. Show active filters as removable chips/tags.

### Overview Tab

```
+------------------+------------------+------------------+------------------+

```

**Unique visitor count note**: Display a subtle tooltip or info icon next to the unique visitors metric noting that unique visitor counts are approximate (based on referrer-domain matching).

### Pages Tab

- Sortable data table: pathname, pageviews, unique visitors, avg duration, bounce rate
- **Entry pages section**: Pages where sessions start (now supported by the data model)
- **Exit pages section**: Pages where sessions end (now supported by the data model)
- Search/filter by pathname

### Referrers Tab

- Referrer domains table: domain, visitors, pageviews
- UTM breakdown:
  - Sources table (utm_source)
  - Campaigns table (utm_campaign)
  - **Mediums table (utm_medium)** - now supported by the data model

### Geography Tab

- Country list with visitor counts and pageview counts
- Sorted by visitor count descending
- Country flags via country code

**Note**: Add a subtle info note or tooltip explaining that country data is derived from browser timezone (not IP geolocation), which means it reflects the user's configured timezone rather than their physical location.

### Devices Tab

- Device type breakdown: desktop, mobile, tablet, other (bar chart or donut)
- Browsers table: browser name, count, percentage
- OS table: OS name, count, percentage
- Screen resolution distribution (if data available)

### Events Tab

- Events summary table: event name, total count, unique visitors
- **Event detail/drilldown view**: Click on an event name to see:
  - Event metadata breakdown (key-value pairs with counts)
  - Event timeseries chart
  - Top pages where the event fires

**Note**: Event metadata keys are validated to match `^[a-zA-Z0-9_]+$` at collection time (see [04-embed-script.md](./04-embed-script.md)). The UI can rely on this constraint when rendering metadata breakdowns.

### Real-time Tab

- **Live visitor count** (large number, updated via WebSocket)
- **Active pages** list (pathname + visitor count, updates live)
- **Live event stream** (scrolling list of recent events/pageviews, last 50)
- **Anomaly indicators** (if any active anomalies detected)

WebSocket connection: authenticated via **first-message pattern** (see [01-architecture.md](./01-architecture.md#websocket-authentication)). The `useAnalyticsWebSocket` hook handles sending the auth token as the first message after connection.

### Settings Tab

- **Embed snippet**: Copyable `<script>` tag and noscript pixel
- **Tracking options**: Toggle scroll depth, session duration, outbound links
- **Ignored paths**: List of path patterns to exclude from tracking
- **Allowed origins**: Additional CORS origins
- **Retention**: Raw event days, aggregate days (ClickHouse TTL as floor; per-domain filtering at query time)
- **Breakdown cardinality cap**: Max entries per breakdown field (default 100, max 500)
- **Public dashboard**: Enable/disable toggle, password setting, shareable URL
- **API token**: Generate/regenerate for server-side event submission
- **Danger zone**: Regenerate UUID (invalidates old scripts), delete all analytics data

## Hooks

**File**: `apps/web/src/hooks/use-analytics.ts`

```typescript
// Extension status
export function useAnalyticsExtensionEnabled(): boolean

// Configuration
export function useAnalyticsConfigs(): UseQueryResult<AnalyticsConfig[]>
export function useAnalyticsConfig(configId: string): UseQueryResult<AnalyticsConfig>
export function useAnalyticsDomainConfig(domainId: string): UseQueryResult<AnalyticsConfig | null>
export function useEnableAnalytics(): UseMutationResult
export function useDisableAnalytics(): UseMutationResult
export function useUpdateAnalyticsConfig(): UseMutationResult

// Data queries (all accept { start?, end?, granularity? })
export function useAnalyticsSummary(configId: string, opts?): UseQueryResult
export function useAnalyticsTimeseries(configId: string, opts?): UseQueryResult
export function useAnalyticsPages(configId: string, opts?): UseQueryResult
export function useAnalyticsReferrers(configId: string, opts?): UseQueryResult
export function useAnalyticsGeography(configId: string, opts?): UseQueryResult
export function useAnalyticsDevices(configId: string, opts?): UseQueryResult
export function useAnalyticsEvents(configId: string, opts?): UseQueryResult
export function useAnalyticsUtm(configId: string, opts?): UseQueryResult
export function useAnalyticsLive(configId: string): UseQueryResult

// Funnels
export function useAnalyticsFunnels(configId: string): UseQueryResult
export function useAnalyticsFunnelResults(funnelId: string, opts?): UseQueryResult
export function useCreateFunnel(): UseMutationResult
export function useUpdateFunnel(): UseMutationResult
export function useDeleteFunnel(): UseMutationResult

// Export
export function useAnalyticsExport(configId: string, type: string, opts?): void

// Real-time WebSocket
export function useAnalyticsWebSocket(configId: string): {
  activeVisitors: number;
  activePages: { pathname: string; visitors: number }[];
  recentEvents: AnalyticsLiveEvent[];
  anomalies: AnalyticsAnomaly[];
  connected: boolean;
}
```

**Query key pattern** (for cache management):

```typescript
export const analyticsKeys = {
  all: ["analytics"] as const,
  configs: () => [...analyticsKeys.all, "configs"] as const,
  config: (id: string) => [...analyticsKeys.all, "config", id] as const,
  summary: (id: string, opts?: DateRange) => [...analyticsKeys.all, "summary", id, opts] as const,
  timeseries: (id: string, opts?: DateRange) => [...analyticsKeys.all, "timeseries", id, opts] as const,
  // ... etc
};
```

## Reusable Components

Leverage existing components from `@uni-proxy-manager/ui`:

- `Card`, `CardHeader`, `CardTitle`, `CardContent` - Metric cards
- `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` - Dashboard tabs
- `Badge` - Status indicators
- `Button` - Actions
- `Skeleton` - Loading states
- `DataTable` - Sortable tables (TanStack React Table)
- `Select` - Period selector
- `Switch` - Toggle settings
- `Input`, `Textarea` - Configuration forms
- `Dialog`, `AlertDialog` - Confirmation dialogs

Charts use **Recharts** (already in the project):
- `LineChart` - Traffic timeseries
- `BarChart` - Device/browser breakdown
- `AreaChart` - Stacked area for event types
- Custom funnel visualisation (see [09-funnels.md](./09-funnels.md))

## Browser Support

The analytics dashboard UI targets modern browsers only (this is the admin-facing dashboard, not the lightweight tracking script). The tracking script has its own broader browser support with graceful degradation -- see [04-embed-script.md](./04-embed-script.md#browser-support).
